# Tourist

Tourist - веб-платформа для подбора туристических маршрутов с персональными рекомендациями. Приложение позволяет зарегистрироваться, настроить профиль, просматривать туристические объекты на карте, получать рекомендованный маршрут, вручную выбирать точки, сохранять маршруты и управлять ими.

## 1. Что нужно установить на пустом ПК

Ниже приведены команды для Windows PowerShell. PowerShell лучше открыть от имени администратора.

Установить Node.js LTS:

```powershell
winget install OpenJS.NodeJS.LTS
```

Установить PostgreSQL:

```powershell
winget install PostgreSQL.PostgreSQL.16
```

Установить Git:

```powershell
winget install Git.Git
```

Установить Visual Studio Code:

```powershell
winget install Microsoft.VisualStudioCode
```

После установки закройте и заново откройте PowerShell, чтобы обновились переменные окружения.

Проверить установку Node.js и npm можно командами:

```powershell
node -v
npm -v
```

Проверить PostgreSQL:

```powershell
psql --version
```

Проверить Git:

```powershell
git --version
```

Если команды не находятся, нужно добавить Node.js или PostgreSQL в переменную окружения `PATH` либо переустановить их с включённой опцией добавления в PATH.

Если `winget` недоступен, установите программы вручную:

- Node.js: `https://nodejs.org/`
- PostgreSQL: `https://www.postgresql.org/download/windows/`
- Git: `https://git-scm.com/download/win`
- Visual Studio Code: `https://code.visualstudio.com/`

## 2. Установка библиотек проекта

Если проект получен в zip-архиве, распакуйте его в удобную папку, например:

```text
C:\tourist
```

Откройте PowerShell в папке проекта или перейдите в неё командой:

```powershell
cd C:\tourist
```

Установите все библиотеки проекта:

```powershell
npm install
```

Эта команда установит библиотеки для клиентской и серверной части, включая React, TypeScript, Express, PostgreSQL-драйвер, Leaflet и остальные зависимости из `package.json`.

Если папка `node_modules` отсутствует или библиотеки были удалены, достаточно снова выполнить:

```powershell
npm install
```

## 3. Настройка базы данных

После установки PostgreSQL убедитесь, что служба базы данных запущена:

```powershell
Get-Service *postgres*
```

Если служба остановлена, запустите её. Название службы может немного отличаться, поэтому сначала посмотрите его через команду выше:

```powershell
Start-Service postgresql-x64-16
```

Создайте базу данных:

```powershell
psql -U postgres -c "CREATE DATABASE tour_routes;"
```

Если база уже была создана раньше, команда может сообщить, что база существует. В таком случае переходите к следующему шагу.

Примените структуру таблиц:

```powershell
psql -U postgres -d tour_routes -f database/schema.sql
```

Добавьте начальные данные:

```powershell
psql -U postgres -d tour_routes -f database/seed.sql
```

Во время выполнения `psql` может запросить пароль пользователя `postgres`. Введите пароль, который был указан при установке PostgreSQL.

## 4. Настройка файла `.env`

Создайте файл `.env` на основе примера:

```powershell
copy .env.example .env
```

На Linux или macOS:

```bash
cp .env.example .env
```

Пример содержимого `.env`:

```env
PORT=4000
DATABASE_URL=postgres://postgres:postgres@localhost:5432/tour_routes
JWT_SECRET=tourist-local-development-secret
CLIENT_ORIGIN=http://127.0.0.1:5173
OPENROUTE_SERVICE_API_KEY=
```

Если пароль PostgreSQL отличается, измените `DATABASE_URL`.

Например, если пароль пользователя `postgres` равен `7777`, строка должна выглядеть так:

```env
DATABASE_URL=postgres://postgres:7777@localhost:5432/tour_routes
```

Параметр `OPENROUTE_SERVICE_API_KEY` можно оставить пустым. В этом случае приложение будет работать, но маршрут будет отображаться прямой линией между выбранными точками. Если добавить ключ OpenRouteService, сервер сможет строить маршрут по дорогам и пешеходным дорожкам.

## 5. Запуск проекта

Запустите клиентскую и серверную часть одной командой:

```powershell
npm run dev
```

После запуска должны быть доступны адреса:

- клиент: `http://127.0.0.1:5173`
- API: `http://127.0.0.1:4000/api`

Проверить работу API можно по адресу:

```text
http://127.0.0.1:4000/api/health
```

Если порт `5173` занят, Vite может запустить клиент на другом порту. В таком случае лучше освободить порт `5173`, потому что сервер разрешает запросы с адреса, указанного в `CLIENT_ORIGIN`.

## 6. Проверка проекта

Для проверки TypeScript можно выполнить:

```powershell
npm run typecheck
```

Для сборки проекта:

```powershell
npm run build
```

## 7. Учётные записи

После применения `database/seed.sql` доступен администратор:

```text
email: admin@tourist.local
пароль: Admin12345
```

Обычный пользователь может зарегистрироваться самостоятельно через форму регистрации.

## 8. Руководство пользователя

### 8.1 Регистрация

Откройте приложение по адресу:

```text
http://127.0.0.1:5173
```

На стартовом экране выберите вкладку регистрации. Укажите имя, email и пароль. После успешной регистрации пользователь автоматически входит в систему.

### 8.2 Вход в систему

Если аккаунт уже создан, выберите вкладку входа, укажите email и пароль. После входа откроется основной интерфейс приложения.

### 8.3 Настройка профиля

Перейдите во вкладку **Профиль**. В профиле можно настроить:

- имя пользователя;
- интересы: история, культура, природа, архитектура, гастрономия и другие;
- бюджет;
- темп путешествия;
- максимальную длительность маршрута.

После изменения настроек нажмите **Сохранить профиль**. Эти данные используются при расчёте персональных рекомендаций.

### 8.4 Работа с картой

Перейдите во вкладку **Карта**. В левой части отображается список туристических объектов, в центре находится интерактивная карта, справа - текущий маршрут.

В списке объектов можно:

- искать объект по названию или описанию;
- фильтровать объекты по категории;
- добавлять объект в маршрут;
- удалять объект из маршрута повторным нажатием.

На карте отображаются маркеры туристических объектов. Выбранные точки выделяются другим цветом.

### 8.5 Персональный рекомендованный маршрут

В блоке **Рекомендовано** отображается персональный маршрут, сформированный на основе профиля пользователя. Для демонстрации автоматический маршрут собирается из туристических объектов Минска, чтобы маршрут был компактным и удобным для просмотра.

Чтобы применить рекомендованный маршрут, нажмите кнопку **Взять**. После этого точки маршрута будут добавлены на карту и в правую панель.

### 8.6 Ручное составление маршрута

Пользователь может самостоятельно добавить точки из списка или с карты. Для сохранения маршрута нужно выбрать минимум две точки.

В правой панели отображаются:

- название маршрута;
- дата маршрута;
- режим передвижения: пешком или на машине;
- количество точек;
- расстояние;
- примерная длительность;
- список выбранных объектов.

Порядок маршрута можно оптимизировать с помощью переключателя **Оптимизировать порядок**.

### 8.7 Сохранение маршрута

После выбора точек укажите название маршрута и при необходимости дату. Затем нажмите **Сохранить маршрут**.

После успешного сохранения приложение переключит пользователя во вкладку **Маршруты**.

### 8.8 Просмотр сохранённых маршрутов

Во вкладке **Маршруты** отображаются сохранённые маршруты пользователя. В карточке маршрута показываются:

- название;
- описание;
- количество точек;
- расстояние;
- длительность;
- список туристических объектов.

Кнопка **Открыть на карте** переносит маршрут обратно в конструктор.

### 8.9 Удаление маршрута

Чтобы удалить маршрут, нажмите кнопку с иконкой корзины в карточке маршрута. После удаления маршрут исчезнет из списка.

### 8.10 Административный раздел

Если пользователь вошёл с ролью администратора, в верхнем меню появляется вкладка **Админ**.

В административном разделе можно:

- просматривать список пользователей;
- видеть количество маршрутов у каждого пользователя;
- просматривать маршруты всех пользователей;
- фильтровать маршруты по выбранному пользователю;
- удалять маршруты.

Обычные пользователи не имеют доступа к административному разделу.

## 9. Частые проблемы

### Не запускается `npm run dev`

Убедитесь, что зависимости установлены:

```powershell
npm install
```

Также проверьте, что Node.js установлен корректно:

```powershell
node -v
```

### Ошибка подключения к базе данных

Проверьте:

- запущен ли PostgreSQL;
- создана ли база `tour_routes`;
- правильно ли указан пароль в `DATABASE_URL`;
- применены ли файлы `database/schema.sql` и `database/seed.sql`.

### Не проходит регистрация или вход

Проверьте, что сервер запущен на `http://127.0.0.1:4000`, а клиент открыт по адресу `http://127.0.0.1:5173`.

Если клиент открылся на другом порту, например `5174`, могут возникнуть проблемы с запросами к API. В таком случае освободите порт `5173` или измените `CLIENT_ORIGIN` в `.env`.

### На карте нет маршрута по дорогам

Если не указан `OPENROUTE_SERVICE_API_KEY`, маршрут отображается прямой линией. Это нормальное резервное поведение приложения.

## 10. Структура проекта

- `src/` - клиентская часть React.
- `src/components/` - компоненты интерфейса.
- `src/lib/` - функции API, форматирования и подписи категорий.
- `server/src/` - серверная часть Express.
- `server/src/routes/` - маршруты API.
- `server/src/repositories/` - работа с базой данных.
- `server/src/services/` - логика рекомендаций и маршрутов.
- `database/schema.sql` - структура базы данных.
- `database/seed.sql` - начальные данные: пользователи, туристические объекты Беларуси и Европы.
- `docs/` - материалы для диплома и презентации.
