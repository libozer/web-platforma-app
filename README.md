# TourMind

Веб-платформа для подбора туристических маршрутов с персональными рекомендациями.

Проект собран под стек из задания на диплом: React, TypeScript, Node.js и PostgreSQL. В MVP реализованы регистрация и вход пользователя, редактирование профиля и предпочтений, интерактивная карта, подбор рекомендованных точек, построение маршрута и сохранение маршрутов в профиль.

## Структура

- `src/` - React-клиент на Vite.
- `server/src/` - Node.js API на Express и TypeScript.
- `database/schema.sql` - структура PostgreSQL.
- `database/seed.sql` - стартовые туристические объекты для Минска.
- `docs/diploma-structure.md` - как реализованный проект закрывает разделы пояснительной записки.

## Запуск

1. Создать базу данных PostgreSQL:

```bash
createdb tour_routes
```

2. Применить схему и тестовые данные:

```bash
psql postgres://postgres:postgres@localhost:5432/tour_routes -f database/schema.sql
psql postgres://postgres:postgres@localhost:5432/tour_routes -f database/seed.sql
```

3. Создать `.env` из примера:

```bash
cp .env.example .env
```

Для построения маршрутов по дорогам можно добавить ключ OpenRouteService в `.env`:

```bash
OPENROUTE_SERVICE_API_KEY=your-api-key
```

Если ключ не указан, приложение продолжит работать и будет показывать маршрут прямой линией между выбранными точками.

4. Установить зависимости и запустить проект:

```bash
npm install
npm run dev
```

Клиент будет доступен по адресу `http://127.0.0.1:5173`, API - по адресу `http://127.0.0.1:4000/api`.

## Основные API

- `POST /api/auth/register` - регистрация пользователя.
- `POST /api/auth/login` - вход.
- `GET /api/profile` - получение профиля.
- `PATCH /api/profile` - обновление профиля и предпочтений.
- `GET /api/attractions` - список туристических объектов.
- `GET /api/recommendations` - персональные рекомендации.
- `POST /api/routes/preview` - расчёт маршрута без сохранения.
- `POST /api/routes` - сохранение маршрута.
- `GET /api/routes` - маршруты пользователя.
- `DELETE /api/routes/:id` - удаление маршрута.

## Логика рекомендаций

Сервис оценивает туристические объекты по интересам пользователя, бюджету, темпу путешествия и рейтингу. Для маршрута применяется простая оптимизация порядка точек методом ближайшего соседа, а итоговая длительность считается как сумма времени посещения и приблизительного времени перемещения между точками.

## Маршрутизация по дорогам

В конструкторе маршрута можно выбрать режим "Пешком" или "На машине". При наличии ключа OpenRouteService сервер получает реальную геометрию маршрута: пеший маршрут отображается зелёной линией, автомобильный - синей. Без ключа используется резервный расчёт по прямой линии.
